# Observability Features — OpenTelemetry + Call Stack Tracing

## Overview

This shared package (@adatechnology/shared) provides two main observability features:

1. **OpenTelemetry RequestId Correlation** — Automatically injects custom requestId into OpenTelemetry spans
2. **Call Stack Tracing** — Automatically tracks method call chains for debugging complex flows

Both features are designed to work together to provide complete request tracing across all services.

---

## Feature 1: OpenTelemetry RequestId Correlation

### Problem

- RequestId (custom 8-char identifier) appears in logs across all services
- RequestId **NOT** visible in Jaeger spans, breaking correlation between logs and traces

### Solution

**OpenTelemetryRequestIdInterceptor** automatically:
1. Reads requestId from request (generated by HttpMetricsInterceptor)
2. Injects into active span as attribute `request.id`
3. Propagates via OpenTelemetry context so libraries auto-share requestId

### Registration

```typescript
// metrics.module.ts (all services)
import { OpenTelemetryRequestIdInterceptor } from '@adatechnology/shared';

@Module({
  providers: [
    {
      provide: APP_INTERCEPTOR,
      useClass: TraceStackInterceptor,  // 1º: Initialize stack
    },
    {
      provide: APP_INTERCEPTOR,
      useExisting: HttpMetricsInterceptor,  // 2º: Generate requestId
    },
    {
      provide: APP_INTERCEPTOR,
      useClass: OpenTelemetryRequestIdInterceptor,  // 3º: Inject into span + context
    },
  ],
})
export class MetricsModule {}
```

### Result

Jaeger now shows spans with `request.id = "O2K2V2YH"` attribute, allowing:
- Click span → View Logs → See all logs from this request in Loki
- Complete correlation: logs ↔ traces ↔ requestId

## Feature 2: Call Stack Tracing

### Problem

Logs show only the current method:
```
[O2K2V2YH][...][CheckEmailExistsUseCase.execute][INFO] Starting check
[O2K2V2YH][...][UserRepository.findByEmail][INFO] Querying database
```

❌ Hard to understand the call hierarchy

### Solution

**TraceStackService + Decorator** automatically track call chains:

```typescript
@Injectable()
export class CheckEmailExistsUseCase {
  constructor(
    private traceStack: TraceStackService,
    private userRepository: UserRepository,
  ) {}

  @TraceMethodWithDI()  // ← Automatic push/pop
  async execute(email: string) {
    // Stack: [CheckEmailExistsUseCase.execute]
    return await this.userRepository.findByEmail(email);
  }
}

@Injectable()
export class UserRepository {
  constructor(private traceStack: TraceStackService) {}

  @TraceMethodWithDI()
  async findByEmail(email: string) {
    // Stack: [CheckEmailExistsUseCase.execute][UserRepository.findByEmail]
  }
}
```

### Result

Logs now show the complete call chain:

```
[O2K2V2YH][...][CheckEmailExistsUseCase.execute][INFO] - Start
[O2K2V2YH][...][CheckEmailExistsUseCase.execute][UserRepository.findByEmail][INFO] - Query
[O2K2V2YH][...][CheckEmailExistsUseCase.execute][UserRepository.findByEmail][INFO] - Result
[O2K2V2YH][...][CheckEmailExistsUseCase.execute][INFO] - Done
```

✅ Immediately clear which methods called which

## API Reference

### TraceStackService

```typescript
import { TraceStackService } from '@adatechnology/shared';

// Get current call stack
stack: string[] = traceStack.getStack();  // ['Method1', 'Method2']

// Get formatted for logs
formatted: string = traceStack.getStackFormatted();  // "[Method1][Method2]"

// Get depth (how many methods deep)
depth: number = traceStack.getDepth();  // 2

// Get current/parent method
current: string = traceStack.getCurrentMethod();  // 'Method2'
parent: string = traceStack.getParentMethod();  // 'Method1'

// Manual push/pop (if not using decorator)
traceStack.push('ClassName.methodName');
try {
  // ... logic
} finally {
  traceStack.pop();
}

// Clear stack (done automatically at request end)
traceStack.clear();
```

### @TraceMethodWithDI() Decorator

Use in any service/class with `TraceStackService` injected:

```typescript
@Injectable()
export class MyService {
  constructor(private traceStack: TraceStackService) {}

  @TraceMethodWithDI()
  async myMethod() {
    // Stack automatically updated on entry/exit
  }
}
```

Works with both sync and async methods. Automatically handles exceptions (pop still called).

To include call stack in all logs, update your log provider:

```typescript

  constructor(
    private logger: LoggerInterface,
    private traceStack: TraceStackService,
  ) {}

  @TraceMethodWithDI()

      requestId: req.requestId,

    });
  }

```

Or create a custom logger wrapper:

```typescript
@Injectable()
export class TracedLogger {
  constructor(
    private logger: LoggerInterface,
    private traceStack: TraceStackService,
  ) {}

  info(message: string, context: string, data?: any) {
    return this.logger.info({
      message,
      context,
      stack: this.traceStack.getStackFormatted(),
      ...data,
    });
  }
}
```

## Integration with Jaeger/OpenTelemetry

Add call stack to span attributes:

```typescript
@Injectable()
export class MyService {
  constructor(private traceStack: TraceStackService) {}

  @TraceMethodWithDI()
  async process() {
    const span = trace.getActiveSpan();
    if (span) {
      span.setAttribute('trace.stack', this.traceStack.getStackFormatted());
    }
  }
}
```

Now Jaeger displays the call stack for each span!

- **Memory:** ~5-10 bytes per method in stack, max ~1KB for 50-method deep call
- **No runtime overhead** if not using decorator

## Troubleshooting

### Stack not appearing in logs

**Issue:** Call stack is tracked but not in log output

**Solution:** Ensure logger includes `stack` field:
```typescript
this.logger.info({
  message: 'test',
  stack: this.traceStack.getStackFormatted(),  // ← Must add this
});
```

### RequestId not in Jaeger spans

**Issue:** RequestId appears in logs but not in Jaeger

**Cause:** OpenTelemetryRequestIdInterceptor not registered or interceptor order wrong

**Solution:** 
1. Verify `OpenTelemetryRequestIdInterceptor` is registered in MetricsModule
2. Verify order: TraceStackInterceptor → HttpMetricsInterceptor → OpenTelemetryRequestIdInterceptor
3. Make request and check Jaeger UI → Service → Search for span → Tags → `request.id`

### Stack not captured in decorator method

**Issue:** @TraceMethodWithDI() used but stack shows empty

**Solution:**
1. Ensure `TraceStackService` is injected: `constructor(private traceStack: TraceStackService)`
2. Ensure `TraceStackInterceptor` is registered (initializes stack per request)
3. Check that HTTP request is actually being processed (stack only works in HTTP context)

## Architecture Diagram

```
HTTP Request arrives
    ↓
TraceStackInterceptor
├─ Clear previous stack (defensive)
├─ Initialize request scope
    ↓
HttpMetricsInterceptor
├─ Generate requestId (8-char nanoid)
├─ Attach to request object
    ↓
OpenTelemetryRequestIdInterceptor
├─ Read requestId from request
├─ Inject into span: span.setAttribute('request.id', 'O2K2V2YH')
├─ Propagate via context (libraries auto-share)
    ↓
Controller / UseCase / Repository
├─ @TraceMethodWithDI() decorator
├─ Automatically push method name on entry
├─ Stack grows as methods call methods
├─ Stack shrinks as methods exit
├─ Logs include: requestId + stack
    ↓
Jaeger (backend)
├─ Receives span with request.id attribute
├─ Receives all spans from this request with same request.id
├─ "View Logs" integration shows logs via Loki
    ↓
Loki (backend)
├─ Logs filtered by request.id
├─ Complete picture of request across all apps
```

## Code Examples

### Example 1: Simple UseCase with Stack

```typescript
import { Injectable } from '@nestjs/common';
import { TraceStackService, TraceMethodWithDI } from '@adatechnology/shared';

@Injectable()
export class CreateUserUseCase {
  constructor(
    private traceStack: TraceStackService,
    private userRepository: UserRepository,
  ) {}

  @TraceMethodWithDI()
  async execute(createUserDto: CreateUserDto) {
    // Stack: [CreateUserUseCase.execute]
    
    const exists = await this.userRepository.findByEmail(createUserDto.email);
    // Stack while executing: [CreateUserUseCase.execute][UserRepository.findByEmail]
    
    if (exists) {
      throw new UserAlreadyExistsException();
    }
    
    return await this.userRepository.create(createUserDto);
    // Stack while executing: [CreateUserUseCase.execute][UserRepository.create]
  }
}
```

### Example 2: Multiple Layers

```typescript
@Injectable()
export class UserController {
  constructor(
    private traceStack: TraceStackService,
    private createUserUseCase: CreateUserUseCase,
  ) {}

  @Post()
  @TraceMethodWithDI()
  async create(@Body() dto: CreateUserDto) {
    // Stack: [UserController.create]
    
    const user = await this.createUserUseCase.execute(dto);
    // Stack while executing: [UserController.create][CreateUserUseCase.execute][UserRepository.create]
    
    return user;
  }
}

@Injectable()
export class UserRepository {
  constructor(
    private traceStack: TraceStackService,
    private dataSource: DataSource,
  ) {}

  @TraceMethodWithDI()
  async findByEmail(email: string) {
    // Stack: [UserController.create][CreateUserUseCase.execute][UserRepository.findByEmail]
    return this.dataSource.query('SELECT * FROM users WHERE email = $1', [email]);
  }

  @TraceMethodWithDI()
  async create(user: CreateUserDto) {
    // Stack: [UserController.create][CreateUserUseCase.execute][UserRepository.create]
    return this.dataSource.query('INSERT INTO users ...', [user]);
  }
}
```

### Example 3: Integration with Logger

```typescript
import { TraceStackService, TraceMethodWithDI } from '@adatechnology/shared';
import { LOGGER_PROVIDER } from '@adatechnology/logger';

@Injectable()
export class UserService {
  constructor(
    @Inject(LOGGER_PROVIDER) private logger: LoggerInterface,
    private traceStack: TraceStackService,
    private repo: UserRepository,
  ) {}

  @TraceMethodWithDI()
  async getUser(userId: string) {
    this.logger.info({
      message: 'Fetching user',
      context: 'UserService.getUser',
      requestId: getUserIdFromContext(),  // From OpenTelemetry context
      stack: this.traceStack.getStackFormatted(),
      userId,
    });

    const user = await this.repo.findById(userId);

    this.logger.info({
      message: 'User fetched',
      context: 'UserService.getUser',
      stack: this.traceStack.getStackFormatted(),
      userName: user.name,
    });

    return user;
  }
}
```
